HTML

  • Enter your HTML notes here
  • The !DOCTYPE html is a declaration letting the brower know to expect HTML on the page.
  • The html tag with an attribute of lang="en" lets brower know that this page will be in English.
  • The head element contains information about the webpage, not visible to the user.
  • The body element represents the visible content shown to the user.
  • The title element defines the title for a webpage.
  • The style element defines teh CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The metal element defines metadata such as teh character set, description, keywords, author, and viewport.
  • The meta tag with charset= element sets our character encoding.
  • The meta tag with http-equiv= element sets an HTTP header for content.
  • The meta tag with name="viewport" element sets the default view for our webpage.
  • The script element embeds client-side scripts int he HTML document.
  • The div element creates a division in the page.
  • The span element creates an inline division.
  • The h element represents the heading with levels for prominent titles and subtitles to organize a page and the order represented in levels matter.
  • The p element represents a paragraph or blocks of text to wrap most of the contents on your webpages.
  • The ul element contains an unordered or bullet point, list of reasons.
  • The ol element represent ordered list.
  • The li element represent list items.
  • The img element contains information about images that are displaye don the webpage.
  • The src element attribute used with img indicate location of the image.
  • The alt attribute used with img contains a text string indicating appearance and functionality of the image.
  • The br element creates a line of empty space or a line break between two blocks of content.
  • The a element, which stands for anchor, creates linkes to the same webpage or other webpage.
  • The href= attribute used with a element points to the URL for the link.
  • The footer element appears at the bottom of the page and contains the author, copyright, contact, sitemap, and navigation.

  • Every HTML element is essentially box. Each box with of margins outside the element, the element’s borders, padding inside the element, and the content inside the padding.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want aroudn the content inside an element.
  • Each property: value is a declaration. The css property we want to apply: and the value of the property.
  • A property can be width, height, background-color, text-align, margin-left, margin-right, padding-left, front-size,etc.
  • A display propert assigned with block value indicates and assigns each image to start on a new line.
  • A link element creates the connection to the external file.
  • A rel attribute specifies the relationship between the current document and the linked document resource.
  • A href attribute specifies the location (URL) of the external resource and is using a relative path.
  • link rel="stylesheet" href="./assets/style.css"
  • A .card designates this as a class selector
  • Any HTML element assigned to the class, card, will gain these property assignments from the CSS.
  • A section class="card" already contains the class attribute. Therefore, all four section elements on the page will contain the same property assignemnts according to the CSS rule.
  • A HTML hexadecimal code defines the color.

Git

  • git status : checks what branch we are currently on.
  • git checkout main: switches to the main branch if you're not there already.
  • git pull origin main : pulls down the latest main branch.
  • git checkout -b feature/branch-name : creates a new branch and switches to it.
  • git add -A: commits all files in the working branch, make sure to check git status to be on the working branch.
  • git commit -m "descriptive commit message of the changes implemented" : commits changes to Git repository up to date with our working directory.
  • git push origin feature/branch-name: to push our changes to the remote GitHub branch, then go to GitHub REPO IN THE BROWSER AND SELECT THE branch-name and select the compare & pull request. Then, add descriptive title with the changes you've made. Then, to main <-- from feature, select create pull request.
  • git pull origin main: to check local branch is in sync after pushing up to GitHub repo
  • git status and git checkout main, repeat.
